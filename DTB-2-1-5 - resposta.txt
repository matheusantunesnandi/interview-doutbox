# 2.1.5 Como podemos gerar rotas automática com Laravel?
Ao invés de definir rotas manualmente usando **Route::get(...)** ou outro verbo, podemos utilizar **Route::resource(...)**.

No momento da criação do controller através do Artisa, podemos usar o argumento **-r** junto para que o controller seja um **resource**, exemplo:

```
php artisan make:controller MeuNovoController -r
```

Após executar o comando, a nova classe MeuNovoController será automaticamente criada com várias funções predefinidas de acordo com o padrão de resource do Laravel.
Virá com funções: index, create, store, edit, update, show e destroy.

Quando usar Route::resource('nome', MeuNovoController::class) as rotas serão automaticamente criadas.

Mais informações de como elas são criadas e como utilizar estão na documentação oficial do Laravel em:
https://laravel.com/docs/11.x/controllers#actions-handled-by-resource-controllers

| Verb | URI | Action | Route |
| -------- | -------- | -------- | -------- |
| GET | /photos | index | photos.index
| GET | /photos/create | create | photos.create
| POST | /photos | store | photos.store
| GET | /photos/{photo} | show | photos.show
| GET | /photos/{photo}/edit | edit | photos.edit
| PUT/PATCH | /photos/{photo} | update | photos.update
| DELETE | /photos/{photo} | destroy | photos.destroy
